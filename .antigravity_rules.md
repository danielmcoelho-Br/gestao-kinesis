# Regras de Conduta - Projeto KinesisLab

As seguintes regras se aplicam a todas as interações com o Antigravity neste projeto:

1. **Idioma de Resposta**: Todas as comunicações e documentações fornecidas pelo Antigravity devem ser em **PORTUGUÊS**, a menos que solicitado de outra forma.
2. **Confirmação de Comandos (Allow running this command?)**: O usuário autoriza e aceitará todas as solicitações de execução de comandos (como o prompt "Allow running this command?"). O Antigravity deve propor e solicitar a execução dos comandos necessários para o desenvolvimento sempre que preciso.
3. **Aprovação de Planos**: O Antigravity deve aguardar a confirmação manual do usuário no chat antes de iniciar a execução de qualquer `implementation_plan.md`. Nenhuma alteração de código ou execução do plano deve ocorrer sem a autorização manual e explícita do usuário.
4. **Proibição de Preview e Navegação Autônoma**: O Antigravity **NUNCA** deve realizar pré-visualizações (previews) no navegador de forma autônoma ou abrir o navegador automaticamente, a menos que solicitado explicitamente pelo usuário. O uso de ferramentas de browser deve ser restrito exclusivamente a pesquisas técnicas externas que não envolvam o sistema em desenvolvimento.
5. **Pesquisa antes da Automação**: Antes de sugerir mudanças complexas, realize uma pesquisa detalhada na base de código para garantir o alinhamento com os padrões existentes.
6. **Redirecionamento de Projetos (KinesisLab vs. kinesis-app)**: Embora a pasta ativa aberta no IDE/editor seja `KinesisLab`, o aplicativo real e servidor de desenvolvimento do usuário é o `kinesis-app` (localizado em `C:\Users\daniel\.gemini\antigravity\scratch\kinesis-app`). Sempre que o usuário solicitar rodar o localhost, salvar relatórios, ou fazer modificações no código da aplicação, o Antigravity deve obrigatoriamente direcionar essas ações e salvar os arquivos correspondentes na pasta `C:\Users\daniel\.gemini\antigravity\scratch\kinesis-app`.
